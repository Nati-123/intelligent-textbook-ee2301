# Priority Encoder Simulator

An interactive 4-to-2 priority encoder that encodes the highest-priority active input.

## Overview

This MicroSim demonstrates how a priority encoder handles multiple active inputs by selecting and encoding only the highest-priority (highest-numbered) active input.

## Learning Objectives

- **Understand** priority encoding when multiple inputs are active (Bloom Level: L2)
- **Apply** priority encoder behavior to interrupt handling scenarios (Bloom Level: L3)

## How to Use

1. **Toggle** multiple data inputs D0-D3
2. **Observe** which input gets encoded (highest active wins)
3. **Note** the Valid output indicating at least one input is active
