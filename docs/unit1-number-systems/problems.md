### Problem 9

b) 35 + (-47)
c) (-35) + (-47)

<details style="margin-top: 1rem;">
<summary style="color: #5A3EED; font-weight: 700; cursor: pointer;">Show Solution</summary>
<div style="background: #E7F7E7; border: 2px solid #81C784; border-radius: 10px; padding: 18px 22px; margin-top: 10px;">
<p style="color: #333; line-height: 1.85; margin-top: 0;"><strong>a)</strong> 35 = 00100011, 47 = 00101111<br>

<p style="color: #333; line-height: 1.85;"><strong>b)</strong> 35 = 00100011, -47 = 11010001<br>
00100011 + 11010001 = 11110100 = <strong>-12</strong> &#10003;</p>
<p style="color: #333; line-height: 1.85;"><strong>c)</strong> -35 = 11011101, -47 = 11010001<br>
11011101 + 11010001 = 110101110 (carry out)<br>
Result = 10101110 = <strong>-82</strong> &#10003;</p>
<p style="color: #333; line-height: 1.85; margin-bottom: 0;"><strong>d)</strong> 100 = 01100100, 50 = 00110010<br>
01100100 + 00110010 = 10010110 = -106<br>
<span style="color: #D32F2F; font-weight: 700;">Overflow!</span> (positive + positive = negative)<br>
Actual answer should be 150, but it exceeds +127</p>

<p style="color: #333; line-height: 1.85; margin-bottom: 0;"><strong>d)</strong> 1111 + 1 = 10000, but only 4 bits, so <strong>0000</strong> (wrap around)</p>

### Problem 19
A computer uses 32-bit addresses.

a) How many bytes of memory can be addressed?
b) Express this in KB, MB, and GB
c) If upgrading to 64-bit addressing, how much more memory can be addressed?

<details style="margin-top: 1rem;">
<summary style="color: #5A3EED; font-weight: 700; cursor: pointer;">Show Solution</summary>
<div style="background: #E7F7E7; border: 2px solid #81C784; border-radius: 10px; padding: 18px 22px; margin-top: 10px;">
<p style="color: #333; line-height: 1.85; margin-top: 0;"><strong>a)</strong> <span class="arithmatex">\(2^{32}\)</span> = <strong>4,294,967,296 bytes</strong></p>
<p style="color: #333; line-height: 1.85;"><strong>b)</strong> 4,294,967,296 bytes = 4,194,304 KB = 4,096 MB = <strong>4 GB</strong></p>
<p style="color: #333; line-height: 1.85; margin-bottom: 0;"><strong>c)</strong> 64-bit: <span class="arithmatex">\(2^{64}\)</span> = 18,446,744,073,709,551,616 bytes = <strong>16 exabytes</strong><br>
Increase factor: <span class="arithmatex">\(2^{64} / 2^{32} = 2^{32}\)</span> = <strong>4 billion times more</strong></p>

### Problem 20
Design a number format for a simple embedded system that needs to represent:

- Temperatures from -40°C to +85°C
- Resolution of 0.5°C

a) How many bits are needed?
b) What encoding scheme would you use?
c) Show the encoding for -10°C, 0°C, and 25.5°C

<details style="margin-top: 1rem;">
<summary style="color: #5A3EED; font-weight: 700; cursor: pointer;">Show Solution</summary>
<div style="background: #E7F7E7; border: 2px solid #81C784; border-radius: 10px; padding: 18px 22px; margin-top: 10px;">
<p style="color: #333; line-height: 1.85; margin-top: 0;"><strong>a)</strong> Range: -40 to +85 = 126 degrees<br>
With 0.5°C resolution: 126 / 0.5 = 252 values needed<br>
<span class="arithmatex">\(2^8 = 256\)</span>, so <strong>8 bits</strong> are sufficient</p>
<p style="color: #333; line-height: 1.85;"><strong>b)</strong> <strong>Offset binary</strong> encoding:</p>
<ul style="list-style: none; padding-left: 0; margin: 0.3rem 0 0.8rem 0;">
<li style="margin-bottom: 0.3rem; line-height: 1.75; color: #333; padding-left: 1.4rem; text-indent: -1.4rem;"><span style="color: #5A3EED; font-weight: 700; margin-right: 0.4rem;">&#9679;</span> Offset = 40 (to make -40°C map to 0)</li>
<li style="margin-bottom: 0.3rem; line-height: 1.75; color: #333; padding-left: 1.4rem; text-indent: -1.4rem;"><span style="color: #5A3EED; font-weight: 700; margin-right: 0.4rem;">&#9679;</span> Value = (Temperature + 40) / 0.5</li>
<li style="margin-bottom: 0.3rem; line-height: 1.75; color: #333; padding-left: 1.4rem; text-indent: -1.4rem;"><span style="color: #5A3EED; font-weight: 700; margin-right: 0.4rem;">&#9679;</span> Or use two's complement with implicit scaling</li>
</ul>
<p style="color: #333; line-height: 1.85;"><strong>c)</strong> Using offset binary (offset = 40, scale = 0.5):</p>
<ul style="list-style: none; padding-left: 0; margin: 0.3rem 0;">
<li style="margin-bottom: 0.3rem; line-height: 1.75; color: #333; padding-left: 1.4rem; text-indent: -1.4rem;"><span style="color: #5A3EED; font-weight: 700; margin-right: 0.4rem;">&#9679;</span> -10°C: (-10 + 40) / 0.5 = 60 = <strong>00111100₂</strong></li>
<li style="margin-bottom: 0.3rem; line-height: 1.75; color: #333; padding-left: 1.4rem; text-indent: -1.4rem;"><span style="color: #5A3EED; font-weight: 700; margin-right: 0.4rem;">&#9679;</span> 0°C: (0 + 40) / 0.5 = 80 = <strong>01010000₂</strong></li>
<li style="margin-bottom: 0; line-height: 1.75; color: #333; padding-left: 1.4rem; text-indent: -1.4rem;"><span style="color: #5A3EED; font-weight: 700; margin-right: 0.4rem;">&#9679;</span> 25.5°C: (25.5 + 40) / 0.5 = 131 = <strong>10000011₂</strong></li>
</ul>
</div>
</details>
